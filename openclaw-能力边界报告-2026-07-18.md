### 1.3 ⭐ 消息生命周期（对记忆系统最有价值）

消息生命周期的完整钩子链路：

```
inbound_claim → message_received → before_prompt_build → before_agent_run
→ llm_input → before_tool_call → after_tool_call → before_agent_reply
→ before_agent_finalize → agent_end → reply_dispatch → message_sending
→ reply_payload_sending → message_sent
```

**关键发现**：
- `message_sending` 和 `reply_payload_sending` 可以在消息发送前拦截/改写——可用于"写记忆前拦截"
- `session_end` + `before_compaction` 是记忆自动存储的最佳 hook 时机
- `heartbeat_prompt_contribution` 专门为 Heartbeat 模式设计

### 2.4 ⭐ 对记忆系统自动化的关键发现

**memory-core 是记忆系统的"核心底座"**，提供：
1. `memory_search` / `memory_get` 工具（已被当前环境启用）
2. 系统提示中的记忆使用指南注入
3. 压缩前自动记忆刷新（Memory Flush）← **这是目前最主要的自动化记忆写入机制**
4. 可选的后台梦见整合（Dreaming）← **opt-in、需要配置**
5. SQLite 索引（FTS5 + Vector）+ 文件监听

**当前环境的记忆自动化状态**：
- ✅ 自动记忆刷新已启用（压缩前触发）
- ❌ 梦见系统未启用（Dreaming disabled by default）
- ❌ Active Memory 子代理未启用（disabled）
- ❌ Memory Wiki 未启用（disabled）
- ✅ 基础记忆搜索和读取可用

## 10. 总结：对记忆系统自动化最有价值的发现

### ⭐ 关键发现 TOP 5

| 排名 | 发现 | 详情 | 行动建议 |
|------|------|------|----------|
| ⭐1 | **Auto Memory Flush 已可用** | 压缩前自动执行静默 LLM turn 写入 `memory/YYYY-MM-DD.md` | 已验证启用；无需额外配置 |
| ⭐2 | **Dreaming 可配置但未启用** | 三阶段梦见系统（Light→REM→Deep）+ Dream Diary | `plugins.entries.memory-core.config.dreaming.enabled: true` |
| ⭐3 | **插件钩子系统可 Hook 消息生命周期** | `session_end` + `before_compaction` +自定义插件 | 可以编写插件在消息发送前/后调用 `memory_store.sh` |
| ⭐4 | **Active Memory 插件存在但禁用** | 在回复前运行阻塞记忆召回子代理 | `plugins.entries.active-memory.enabled: true` |
| ⭐5 | **session-memory hook 已内置** | `/new` 和 `/reset` 时将最后 N 条消息写入 `memory/` | 可通过 `openclaw hooks enable session-memory` 启用 |

### 当前记忆系统自动化状态图

```
用户对话 → memory_search/memory_get ← memory-core 索引
  ↓
压缩前 → Auto Memory Flush → 写入 memory/YYYY-MM-DD.md [✅ 启用]
  ↓
Dreaming (Light→REM→Deep) → 提升至 MEMORY.md [❌ 未启用]
  ↓
Active Memory 子代理 → 注入记忆到上下文 [❌ 未启用]
```
